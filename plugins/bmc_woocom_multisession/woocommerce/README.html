Description

    Modifies WooCommerce's one user account to one shopping cart paradigm to provide for different people logging into the same user account to have multiple shopping carts, shipping & billing addresses and order histories separate from each other.

    Possible scenarios that this plugin addresses are:

    • A site has or anticipates having more than one customer per user account login and wants to keep the shopping carts for each of these customers separate from the other.
    • Users of a site regularly select several distinct orders that are billed separately or are shipped to different locations and prefer not to log into more than one user account to do this.

    Originally developed for B2B situations whereby workers at different offices may all utilize the same vendor account without affecting each others' stored shopping carts.

    Features

    • Use up to ten (10) shopping carts.
    • Conveniently manage and switch between multiple carts from the Wordpress Admin Toolbar or WooCommerce Multiple Carts widget.
    • *NEW* Set carts to Storage Bin mode to keep items from being emptied after checkout.
    • Copy items and their quantities from other carts to the currently active cart.
    • Clone carts including items and addresses.
    • Separate account addresses, downloads and order history per cart.
    • Integration with WooCommerce Multiple Addresses plugin.
    • Organize carts with customized names.
    • Link carts sessions across browsers and devices via key phrase or per user account.

    Requirements

    • WooCommerce 2.3+ (may work for earlier versions but untested)
    • PHP 5.3+

    Installing

    1. If upgrading or otherwise reinstalling this plugin, deactivate and uninstall any existing version of this plugin first.
    2. Go to: Step by Step Guide to Install a WordPress Plugin for Beginners
    3. Scroll down to section Install a Plugin using Upload Method in WordPress Admin Panel and follow the instructions.

    The plugin takes instant effect when activated.

    Uninstalling

    Checking the Delete Data On Uninstall setting will remove the multiple shopping carts generated by this plugin (which can add up over time) and settings. If you are uninstalling this plugin in order to install a newer version, do not check this setting.

    Users may need to empty their shopping carts when this plugin has been deactivated for WooCommerce to reset back to the default single shopping cart per user behavior.

    Considerations

    Cart User Accounts

    This plugin tells WooCommerce / Wordpress to save shopping cart data under fake user account ids. These ids are typically very large numbers and it is assumed that kind of sites using this plugin will have far fewer user accounts; however, in the unlikely event that the fake user account id corresponds with an actual user account id, this plugin will overwrite the shopping cart data for that user.

    WooCommerce Templates

    Turning off the Customer Account Access setting will override the default My Account page template provided by WooCommerce if the current theme does not already use a custom My Account page template. If it does then it is up to the developer of your theme to customize the template to remove links to the Edit Account Details page from the template.

    New Carts

    Creating a New Cart will automatically select that cart.

    Deleting Carts

    Deleting a cart will move any orders associated with that cart to the current cart.

    WooCommerce Required

    This plugin will automatically deactivate if WooCommerce is not activated.
